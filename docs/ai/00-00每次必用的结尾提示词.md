从现在开始，请在每一次任务执行完成后，**强制执行以下收尾步骤**：

1. 更新 `/CHANGELOG.md`
   - 仅记录「发生了什么」
   - 包括：新增 / 删除的实体、路径、语义或架构层面的可观测变化
   - 不解释原因，不记录过程，不列出具体文件名

2. 生成或覆盖 `/CHANGE_SUMMARY.md`
   - 用于描述“本次修改的执行摘要”
   - 必须包含以下小节：
     - Purpose（本次修改的目标）
     - Nature of Change（修改性质：文档 / 架构 / 功能 / 修复）
     - Affected Files（涉及的文件名列表）
     - Added / Removed（新增或删除的文件，文件名 + 简述）
     - Current Status（当前是否稳定，是否存在已知问题）

3. 在生成上述两个文件后：
   - 明确告知我已完成 CHANGELOG 与 CHANGE_SUMMARY 的更新
   - 不将 CHANGE_SUMMARY 视为规则或宪法的一部分

如果本次任务未产生任何可记录的变化，请明确说明“无需更新日志”。
