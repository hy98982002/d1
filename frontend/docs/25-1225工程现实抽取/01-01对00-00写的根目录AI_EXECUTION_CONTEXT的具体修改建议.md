# 一、哪些句子需要改语气（逐条点名）

下面我按 **“危险程度”** 分级标注。

---

## 🔴 高危（会被 LLM 当成「次级宪法」）

这些句子**必须改**，否则 ClaudeCode/Trae 会无意识“执法”。

### ① 阶段体系（5.1）

**原文：**

> 禁止新增旧阶段或引入旧 → 新映射逻辑

**问题：**

* 出现 **「禁止」**
* 是强约束语言
* AI 会认为这是**不可违反规则**

**真实定位应是：**

> 当前代码依赖这一假设，但它不是宪法

---

### ② Slug 规则（5.2）

**原文：**

> slug.ts 是 URL 的唯一权威来源（single source of truth）

**问题：**

* “唯一权威来源”= 法律语言
* 这已经不是“背景”，而是“裁决标准”

---

**原文：**

> Level 要求：必须使用系统级术语

**问题：**

* “必须”= 执行级命令
* ClaudeCode 会用它反向否定代码变更

---

### ③ JSON-LD（5.3）

**原文：**

> JSON-LD 覆盖 Level / Type / Access / Outcome / Pathway 五维字段

**问题：**

* 看起来像“模型定义完成”
* AI 会默认这是**稳定 schema**

但你自己很清楚：
👉 **这是当前实现，不是最终结构**

---

### ④ 开发标准（7 / 10 章）

**原文：**

> 仅使用 Composition API，严格禁止使用 Options API
> 永远不要：使用 Options API
> 始终遵循：

**问题：**

* 「严格禁止 / 永远不要 / 始终遵循」
* 100% 规范语言
* **已经完全越界成执行细则 / 宪法附录**

---

### ⑤ 部署与构建（8 章）

**原文：**

> 构建命令：`npm run build`

**问题：**

* 不仅是规则
* 还是 **错误规则（你用 pnpm）**
* 这类内容**根本不该出现在这个文档**
