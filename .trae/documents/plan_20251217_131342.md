# 修改课程slug的解决方案

## 问题分析

### 核心问题
用户想把所有模拟课程的slug从`-basic`改为`-beginner`，但担心这会影响`educationLevel`（LRMI）的生成。

### 关键发现
1. **educationLevel的生成机制**：
   - `educationLevel`是从`course.stage`字段生成的，不是从`course.slug`提取的
   - 直接使用`StageMeta[course.stage].label`生成中文标签
   - 与`course.slug`没有直接关系

2. **slug与stage的对应关系**：
   - `course.stage: 'basic'` → `StageMeta[course.stage].slug: 'beginner'`
   - 这意味着`basic`阶段对应的slug应该是`beginner`

3. **当前问题**：
   - 模拟课程的slug使用了`-basic`后缀
   - 但根据`StageMeta`配置，正确的后缀应该是`-beginner`

## 解决方案

### 1. 只修改slug中的后缀，不影响educationLevel

#### 结论
✅ **只修改`courseStore.ts`中的slug后缀是安全的**，因为：
- `educationLevel`是从`course.stage`生成的，不是从`course.slug`提取的
- `buildCourseJsonLd`函数中的`educationalLevel`字段是从`course.stage`生成的
- 修改`course.slug`不会直接影响`educationLevel`的生成

#### 具体操作
1. 遍历`courseStore.ts`中的所有课程
2. 将slug中的`-basic`替换为`-beginner`
3. 保持`course.stage`字段不变

### 2. 示例修改

**修改前**：
```typescript
{
  id: 1,
  title: 'Photoshop AI设计技能体验课：从小白到高手',
  slug: 'photoshop-ai-design-basic', // 旧slug
  stage: 'basic' // 阶段字段保持不变
}
```

**修改后**：
```typescript
{
  id: 1,
  title: 'Photoshop AI设计技能体验课：从小白到高手',
  slug: 'photoshop-ai-design-beginner', // 新slug
  stage: 'basic' // 阶段字段保持不变
}
```

### 3. 预期效果
- ✅ 所有课程slug统一使用`beginner/intermediate/advanced`后缀
- ✅ 符合AEO/LRMI最佳实践
- ✅ educationLevel生成不受影响
- ✅ 保持前后端一致性

## 实施步骤

1. 打开`courseStore.ts`文件
2. 遍历所有课程对象
3. 使用正则表达式或字符串替换将`-basic`替换为`-beginner`
4. 验证所有slug修改正确
5. 运行类型检查确保没有错误

## 最终结论

**是的，你可以安全地将所有课程slug中的`-basic`改为`-beginner`**，这不会影响`educationLevel`的生成，因为：
- `educationLevel`是从`course.stage`生成的，不是从`course.slug`提取的
- 修改slug只是保持与`StageMeta`配置的一致性
- 符合项目宪法中关于slug使用系统定义术语的要求

这个修改是安全的，并且符合AEO/LRMI最佳实践！
